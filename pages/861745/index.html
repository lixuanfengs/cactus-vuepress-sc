微服务Docker化

在第五章的5.4小节中，我们通过Vagrant构建了一台内存为6GB，CPU核心数为2，IP地址为192.168.33.10的CentOS 7虚拟机，并且安装好了Docker，Docker Compose。这节我们将继续使用该虚拟机来演示如何将我们的微服务Docker化。

# 微服务改造

我们在第一章搭建认证服务器的时候，在febs-common模块里引入了spring-boot-starter-data-redis依赖，该依赖包含Redis自动装配功能。febs-server-system、febs-server-test和febs-gateway模块都引用了febs-common模块，所以这三个模块也具有Redis自动装配功能。但这三个模块到现在为止并没有使用到Redis，所以我们需要在febs-server-system、febs-server-test和febs-gateway模块的配置文件里添加如下内容，关闭Redis自动装配：

spring:
  autoconfigure:
    exclude: org.springframework.boot.autoconfigure.data.redis.RedisAutoConfiguration
1
2
3

目前，我们在微服务配置文件里配置的地址都是直接使用localhost或者127.0.0.1指定，我们需要将它们改为变量的形式，方便后期使用Docker Compose联调。

在febs-auth模块的配置文件applicaiton.yml里，我们分别配置了febs-monitor-admin、MySQL数据库、Redis和febs-register的地址，现在将这些地址改为变量的形式，改造后配置文件内容如下所示：

上面脚本包含4条命令：

  1. FROM openjdk:8u212-jre表示由openjdk:8u212-jre基础镜像构建。因为我们的项目使用的是JDK 1.8，所以我们要依赖于1.8版本的JDK镜像构建，openjdk官方Docker镜像仓库为https://hub.docker.com/_/openjdk?tab=tags，我挑了个体积相对较小的openjdk:8u212-jre；
  2. MAINTAINER MrBird 852252810@qq.com指定镜像的作者及联系方式；
  3. COPY febs-auth-1.0-SNAPSHOT.jar /febs/febs-auth-1.0-SNAPSHOT.jar表示将当前目录（/febs/febs-auth）下的febs-auth-1.0-SNAPSHOT.jar拷贝到openjdk:8u212-jre镜像里的/febs目录下，名称也为febs-auth-1.0-SNAPSHOT.jar；
  4. ENTRYPOINT ["java", "-Xmx256m", "-jar", "/febs/febs-auth-1.0-SNAPSHOT.jar"]表示运行java -jar运行镜像里的jar包，JVM内存最大分配为256m（因为要运行的微服务较多并且虚拟机内存只有6GB，所以内存分配小了点，实际可以根据宿主服务器的配置做出相应调整）。

DockerFile创建好后，在/febs/febs-auth目录下运行以下命令构建镜像：

docker build -t febs-auth .
